# Docker 容器排障速记

## 来源
- 演示用 inbox 输入
- 场景：本地启动容器后，想快速看日志、进容器、确认端口映射

## 结论
- 查问题时优先看日志，再确认容器状态和端口映射
- 容器里如果没有 `bash`，通常改用 `/bin/sh`
- `docker compose` 管理的服务，排障时优先使用 `docker compose ps` 和 `docker compose logs`

## 适用场景
- 容器启动后访问不到服务
- 想确认容器里程序有没有正常启动
- 想进入容器检查配置文件或运行环境

## 步骤
### 1. 查看容器日志
```bash
docker logs 容器名
docker logs -f --tail 100 容器名
```

### 2. 查看容器状态
```bash
docker ps
docker ps -a
```

### 3. 进入容器内部
```bash
docker exec -it 容器名 /bin/sh
docker exec -it 容器名 /bin/bash
```

### 4. 查看容器详情和端口映射
```bash
docker inspect 容器名
docker port 容器名
```

### 5. 如果是 docker compose 项目
```bash
docker compose ps
docker compose logs -f 服务名
```

## 坑点
- `Exited` 状态说明容器已经退出，先查日志再决定是否重启
- 某些精简镜像没有 `bash`，直接执行会报错
- 端口没映射到宿主机时，容器内部服务正常也无法从本机访问
